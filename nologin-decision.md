# nologin decision

We choose `/usr/sbin/nologin` instead of `/bin/false` for the KijaniKiosk service accounts because it provides clearer operational semantics and better administrator feedback while still preventing interactive logins. Both shells prevent user access, but `/usr/sbin/nologin` returns a helpful message ("This account is currently not available") when a login attempt is made, making troubleshooting easier for system administrators and auditors reviewing access controls. In contrast, `/bin/false` silently exits and may appear as an authentication or configuration failure rather than an intentional security restriction. Using `/usr/sbin/nologin` aligns with common Linux hardening guidance for service accounts by explicitly documenting that the account is non-interactive while maintaining least privilege principles.
